# Screenshot Plan

Screenshots should show real Beta Release B UI and safe demo data. Do not use private customer data. Do not show only login, splash, or title screens.

## Apple
- iPhone 6.9 inch set recommended.
- iPad 13 inch set only if iPad support is enabled and tested.
- Screenshots should communicate the core app experience.

## Google Play
- Phone screenshots required.
- 7-inch tablet screenshots optional if tablet support is listed.
- 10-inch tablet screenshots optional if tablet support is listed.
- Feature graphic required/planned.

## Screenshot Sequence
1. Home - live operations dashboard.
2. Clock - start shift.
3. Clock - active shift.
4. Schedule - upcoming tasks.
5. Timesheets - labour hours/costs.
6. Photos - job-site media.
7. Receipts - expenses by project.
8. Team/Menu - workforce/admin tools.

## Overlay / Caption Ideas
- Track crew hours in real time.
- Clock in from the job site.
- Manage schedules and timesheets.
- Capture receipts and photos.
- Keep field teams organized.

## Demo Data Rule
Use safe demo company data only. Do not create fake production customer data beyond approved demo records.
